# 要件定義書(社内掲示板システム)

## 1. プロジェクト概要
本プロジェクトは、社内の情報共有を効率化するための掲示板システムの開発を目的とする。
本システムにより、スレッド形式で情報を整理・共有し、必要な情報へ迅速にアクセスできる環境を提供する。  
また、ユーザー同士のコミュニケーションを活性化し、業務効率の向上を図る。

---

## 2. スコープ

### スコープ内(実装対象)
#### 閲覧機能
#### 投稿編集機能
#### 便利機能
#### 管理機能

- ユーザー登録・ログイン・ログアウト
- スレッド(投稿)およびレスポンス(返信)の作成・編集・削除
- スレッドの一覧表示
- スレッド内のレスポンス一覧表示
  - レスポンスの上限数：500件
- スレッド名のキーワード検索
- ユーザー名によるスレッド検索
- レスポンスの宛先(レスポンス番号)の指定・表示(アンカー機能)
- スレッドおよびレスポンスへのリアクション機能
- 管理者によるユーザー管理(投稿の追加・編集・削除)

### スコープ外(対象外)
- 画像・PDFファイルの投稿
  - 外部アップローダーサービスで代替
- レスポンス上限到達時の次スレッド自動作成・移行
- スレッド外からのレスポンス検索
- ユーザー名指定のメンション機能
- 匿名投稿機能

---

## 3. 機能要件

### 一般ユーザー向け機能
- ユーザー登録、ログイン、ログアウトができる
- スレッドの作成ができる
- スレッド一覧を閲覧できる
- スレッドに対してレスポンス(返信)を投稿できる
- スレッド内のレスポンス一覧を閲覧できる(最大500件)
- 自身の投稿(スレッド・レスポンス)を削除できる
- スレッド名によるキーワード検索ができる
- ユーザー名によるスレッド検索ができる
- レスポンス番号を指定したアンカー投稿ができる
- スレッドおよびレスポンスにリアクションができる

### 管理者向け機能
- ユーザーの管理(作成・編集・削除)
- 全ユーザーの投稿(スレッド・レスポンス)の管理(追加・編集・削除)

---

## 4. 非機能要件

### 性能
- 同時接続ユーザー数: 5人(将来的には200人程度を想定)
  

### セキュリティ
- XSS 対策
- SQL インジェクション対策
- CSRF 対策
- 認証が必要なページには未ログイン時のリダイレクトを実装

### 対応環境
- PCのみ
  - 対応ブラウザ
    - Firefox
    - Chrome

---

## 5. 用語定義

| 用語 | 定義 |
|------|------|
| スレッド | ユーザーが作成する投稿単位。議題や話題のまとまり |
| レスポンス | スレッドに対する返信投稿 |
| アンカー | 特定のレスポンス番号を指定して参照する仕組み |
| リアクション | 投稿に対して感情や評価を示す機能(いいね等) |
| ユーザー | システムを利用する社員 |
| 管理者 | ユーザーおよび投稿を管理できる権限を持つユーザー |
